Smart Shuffle player

What is it?

The problem with today’s music player applications is that their shuffle algorithms are random which means that people are not getting the best listening experience they could from their music.

What does it do?

This project aims to improve that by quantifying the user’s preference on a basic level: if they listen to it a lot, they must like it a lot.

The application can be installed on any smartphone running Android 5+. The logic was written in Java, using the Android API and the User Interface is built from XML files, which are parsed and compiled to Java objects. Currently, there are a total of 2423 lines of code, 1940 of which are Java code, 483 lines are XML code (layout and the Manifest file).

How does the algorithm work?

Aside from the arcane incantations and the Java-mancy?

It's pretty simple:

  • The algorithm assigns a "weight" to each song in your library (in part random, in other part based on how many times you listened to each song).
  • It puts all the weights in a composite data structure, mapping the songs to their weights.
  • When you request a new song, a random number is generated and used to pick a song; kind of like throwing a dart: the bigger the song's weight, the higher chances it has to be picked.
